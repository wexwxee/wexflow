# Job Apply Hub — как всё устроено и как добавить новую компанию

## Состав

| Что | Где | Порт |
|---|---|---|
| Salling Jobs (дашборд вакансий + бот подачи) | `C:\saling` | 8000 |
| 7-Eleven Apply Studio (заполнитель анкеты) | `C:\seven11-apply` | 7111 |
| Hub — единый вход и переключатель | `C:\saling\hub.py` | **8080** ← открывать это |

Запуск всего сразу: `C:\saling\JobApplyHub.bat` (или ярлык Job Apply Hub).

## Общий дизайн

- Дизайн-система — один файл `theme.css` с тёмной и светлой темами
  (переключатель ☀️/🌙 в шапке, выбор общий для всех приложений через cookie).
- **Источник правды: `C:\saling\static\theme.css`.**
  Копия для 7-Eleven: `C:\seven11-apply\web_static\theme.css` — после правок копируй туда.
- Каждое приложение задаёт свой акцентный цвет переменной `--accent`
  (Salling — синий, 7-Eleven — зелёный).

## Свежесть данных

- Salling: вакансии тянутся из публичного поискового API сайта; обновление
  каждые 30 минут автоматически, индикатор «обновлено N мин назад» в шапке.
- 7-Eleven: список магазинов сканируется прямо с анкеты (Typeform);
  при старте панели база старше 7 дней пересканируется автоматически,
  возраст базы виден в шапке.

## Как добавить новую компанию

Два сценария:

1. **У компании есть доска вакансий** (как у Salling): пишется новый скрейпер
   по образцу `scraper.py` + бот подачи по образцу `apply.py`. Если вакансий
   много и нужен дашборд — проще всего скопировать структуру `C:\saling`
   в новую папку и заменить источник данных.
2. **У компании одна анкета** (как у 7-Eleven): копируется структура
   `C:\seven11-apply`, переписываются правила заполнения (`core/mapping.py`).

Затем:

- подключи новое приложение к хабу: добавь порт в `BACKENDS` в `C:\saling\hub.py`
  и кнопку в переключатель `app-switch` (макрос в `C:\saling\templates\_ui.html`
  и шапка `C:\seven11-apply\web_static\index.html`);
- подключи `theme.css` — дизайн совпадёт автоматически;
- добавь запуск в `C:\saling\Launch-JobApplyHub.ps1` (по образцу блока 7-Eleven).
